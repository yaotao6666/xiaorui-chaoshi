## 3. 验收重点

### 3.1 商家端

- 能正常登录与退出
- 能维护商品、分类、配送、设置
- 商家配送设置页可独立回显并保存 `takeout_enabled`、`dine_in_enabled`、`pickup_enabled`
- 能查看订单详情并完成核销
- 能对已支付 / 已完成订单发起退款
- 工作台今日概览中的待核销订单与已上架商品数口径正确
- 商家分析页库存预警看板支持空态与风险分组展示
- 商家订单列表支持快捷范围和自定义日期筛选
- 商家设置页可进入分账历史，查看日期、金额、比例、状态与失败原因
- 商家设置页以顶部信息卡和功能宫格展示高频入口
- 商家满减规则支持保存、回显、启停和最多 5 档限制
- 商家打印机支持新增、编辑、删除、状态开关、自动打印、默认打印机和测试打印

### 3.2 用户端

- 能扫码进店
- `pages/store/home`、`pages/store/product`、`pages/store/confirm` 支持 `merchant_id` 与 `scene` 两种入口参数
- 能自动登录并完成埋点
- 店铺首页与确认订单页的公开数据请求不依赖先完成登录
- 确认订单页只展示商家已开启的配送 / 堂食 / 自提方式，未开启方式不展示
- 确认订单页可展示满减规则、已命中优惠和下一档提示
- 创建订单后的 `discount_amount`、`pay_amount` 与确认页展示口径一致
- 能浏览商品、加购、下单、支付
- 能查看订单并申请退款

### 3.3 服务商端

- 能直接创建商家
- 能查看商家列表与详情
- 能维护商家 Logo / 背景图
- 能为商家配置 `sub_mch_id`、分账开关与分账比例
- 能查看服务商数据分析
- 能查看服务商分账历史

### 3.4 支付与分账

- 不同商家下单时能拉起各自的 `sub_mch_id`
- 未完成支付配置的商家会被下单拦截
- 使用 `1112649854` 的商家样例可正常拉起支付
- 当前微信小程序登录统一使用 `WECHAT_APP_ID + WECHAT_APP_SECRET`，不再验收双小程序身份切换
- 支付回调后订单支付状态正确
- 分账成功 / 失败 / 跳过三类记录都会写入历史
- 商家端与服务商端都能查询到分账历史

## 4. 推荐测试链路

### 4.1 完整经营链路

1. 商家登录
2. 创建商品与规格
3. 用户扫码进店
4. 店铺首页、商品详情、确认订单按统一入口参数加载当前商家数据
5. 用户完成下单支付
6. 商家查看新创建订单并完成后续处理
7. 商家查看订单详情并核销
8. 服务商查看对应商家分析数据

### 4.2 支付配置与分账链路

1. 服务商创建商家
2. 回填已进件商家的 `sub_mch_id`
3. 配置分账开关与抽佣比例
4. 用户完成下单支付
5. 支付回调更新订单状态
6. 系统立即发起分账或记录跳过原因
7. 服务商与商家查看分账历史

### 4.3 文档链路

1. 先读 `PRD.md`
2. 再读功能说明
3. 再读接口文档
4. 最后读测试与附录

## 6. 部署与环境附录

### 5.1 小程序环境

- HTTP / WS 地址统一走环境配置
- 生产环境需显式配置 `https://` 与 `wss://`

### 5.2 后端环境

- 推荐使用环境变量注入
- 生产环境关闭调试模式
- 敏感配置不落仓库
