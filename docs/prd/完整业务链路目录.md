# 完整业务链路目录

## 1. 文档定位

本文件用于按角色梳理完整业务链路，方便研发、测试、产品和大模型按顺序理解系统主流程。

## 2. 商家经营链路

1. 商家登录
2. 店铺资料准备
3. 分类与商品配置
4. 配送与营业状态设置
5. 查看分账历史
6. 接收订单提醒
7. 查看订单详情
8. 核销订单
9. 处理退款
10. 查看经营分析

## 3. 用户下单链路

1. 扫码进入店铺
2. 自动登录与埋点
3. 浏览商品与分类切换
4. 选择规格与数量
5. 加入购物车
6. 提交订单
7. 微信支付
8. 查看订单
9. 申请退款

## 4. 支付回调链路

1. 微信支付回调进入服务端
2. 校验签名并解密回调报文
3. 更新订单支付状态与交易单号
4. 判断商家分账开关与比例
5. 发起分账或记录跳过原因
6. 更新订单分账状态
7. 写入商家分账抽佣历史

## 5. 服务商商家配置链路

1. 服务商登录
2. 直接创建商家与商家账号
3. 维护商家基础资料与图片
4. 回填已进件商家的 `sub_mch_id`
5. 配置分账开关与抽佣比例
6. 查看支付配置状态
7. 查看商家经营数据
8. 在 `web-admin` 订单管理页按商家筛选查看订单并进入详情页
9. 在 `web-admin` 公告管理页发布系统公告

## 6. 分账链路

1. 商家支付成功
2. 系统按商家配置判断是否分账
3. 生成分账单号
4. 记录抽佣比例、抽佣金额、商家实收
5. 记录成功 / 失败 / 跳过状态
6. 服务商查看分账历史
7. 商家查看本店分账历史

## 7. 验证建议

- 每次完整回归至少覆盖：
  - 商家经营链路
  - 用户下单链路
  - 支付回调链路
  - 服务商商家配置链路
  - 分账链路
